# 描述  
我们在网上搜教程、资料时，有时候会搜到一些大佬的博客，里面记录的他们所分享的一些技术和日常，这时候应该也会想拥有一个属于自己的博客，这篇教程就是手把手教你如何制作一个属于自己的博客

---

# 第一步：准备工作

1、注册GitHub账号  
前往GitHub官网（`https://github.com/`）注册一个GitHub账号,注册成功后记住**用户名**，如果不记得也可以到官网这里查看![GitHub用户名](/assets/images/2025-12-01/1-1.png)

2、下载git  
去git官网（`https://git-scm.com/`）下载![git官网](/assets/images/2025-12-01/1-2.png)

3、下载vscode  
去vscode官网（`https://code.visualstudio.com/`）下载![vscode官网](/assets/images/2025-12-01/1-5.png)

---

# 第二步：创建GitHub仓库

1、登录 GitHub → 点击右上角 → Repository → New![创建仓库1](/assets/images/2025-12-01/1-3.png)

2、仓库名建议用：<你的GitHub用户名>.github.io（这是 GitHub Pages 默认的个人博客仓库名）

3、选择 Public，勾选 Initialize this repository with a README

4、点击 Create Repository![创建仓库2](/assets/images/2025-12-01/1-4.png)

---

# 第三步：启动GitHub Pages(生成博客)

1、进入仓库 → Settings → Pages<br>

配置：

Source：Deploy from branch

Branch：main /root
![启动GitHubPages](/assets/images/2025-12-01/1-8.png)

---

# 第四步：clone远程仓库到本地

1、创建一个文件夹专门放你的项目，使用vscode打开

2、去到你的github的blog仓库，复制仓库的URL![复制仓库URL](/assets/images/2025-12-01/1-6.png)

3、在vscode里面点击**克隆git仓库**，将复制的仓库URL输入进去![clone远程仓库](/assets/images/2025-12-01/1-7.png)

---

# 第五步：正式开始制作blog

1、首先这是一个最小的blog所需要的文件内容
```markdown
your-blog/                                      --你clone的仓库文件夹
├─ _config.yml                                  --站点配置文件                                          
├─ Gemfile(可选)                                 --用于本地运行博客，若有需求可以看我上一篇blog
├─ index.md                                     --站点首页文件
├─ about.md(可选)                                --介绍你自己的页面
├─ archive.md(可选)                              --归档页面，用来显示以往所有的blog的列表      
├─ _posts/                                      --blog文件夹，存放你的blog文章
│   └─ 2025-12-01-your-first-post.md
├─ assets/(可选)                                     --图片文件夹，存放照片，路径不能少
│   └─ images/
│       └─ avatar.png
│       └─ xxx.png
```

2、先在项目根目录创建`_config.yml`文件，以下是示例配置：
```markdown
title: 后果的博客                                 --网站标题
description: 我的学习记录                         --描述你的网站
theme: minima                                   --网站主题，这是GitHub的官方主题，需要配置第三方主题的需要其他方法
markdown: kramdown                              --指定指定 Markdown 渲染器
future: true                                    --允许生成未来日期文章（不能漏，我遇到个情况就是写当天日期的文章上传无显示）
```

3、在项目根目录创建首页文件`index.md`，以下是示例代码：
```markdown
---
layout: home                                    --页面布局，跟主题有关
title: 首页                                      --首页标题
---

欢迎来到我的博客！                                 --正文内容

```html
<h2>文章列表</h2>
<ul>
  {% for post in site.posts %}
    <li>
      <a href="{{ post.url }}">{{ post.title }}</a> - {{ post.date | date: "%Y-%m-%d" }}
    </li>
  {% endfor %}
</ul>                                           --因为GitHub自己的主题很简单，想要主页列举文章列表就需要这堆代码
```
```

4、在项目根目录创建`_posts`文件夹：
```powershell
mkdir _posts
```
在`_posts`里面创建一个示例文章，文件名写：
```text
2025-12-01-hello-world.md
```
内容写：
```markdown
---
layout: post
title: "Hello World"
date: 2025-12-01                                --文章时间
---

这是我的第一篇文章！

![描述](相对路径)                                 --若要使用图片可以这样
![搜索git](/assets/images/2025-12-01/1-9.png)

```
**注意：文件名格式必须是YYYY-MM-DD-title.md，也就是year4位，month2位，day2位，错误示例：2025-12-1-my-first.md**

# 第六步 提交到远程GitHub仓库

1、初始化git(首次使用才需要)<br>
如果是第一次使用 Git，需要把你的身份告诉 Git：
```bash
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub绑定邮箱"
```
只需要设置一次，以后不用再设置。

2、进入你的博客根目录<br>
有两种方法：第一种是在windows自带的搜索里面搜索git![搜索git](/assets/images/2025-12-01/1-9.png)<br>

然后使用使用`cd`进入项目根目录，例如：`cd E:\ITEM\taciturn-hg.github.io`<br>

第二种方法就是进入你项目根目录文件夹，右键 → 显示更多选项 → Open Git Bash Here![打开git bash](/assets/images/2025-12-01/1-10.png)<br>

3、查看当前文件夹修改情况
```bash
git status
```
你会看到红色和绿色的文件，表示哪些被修改/添加了。

4、把修改加入暂存区

把所有改动添加进去：
```bash
git add .
```
或者你想添加某一个文件：
```bash
git add _posts/2025-12-01-第一篇博客.md
```

5、提交代码

加上自己的说明（提交信息尽量写清楚）：
```bash
git commit -m "修改文章：修复文件名日期格式"
```
当你看到：
```bash
X files changed ...
```
说明 commit 成功。

6、推送到 GitHub

把本地提交推送上去：
```bash
git push
```
如果第一次 push，可能需要：<br>
`git push -u origin main`或`git push -u origin master`<br>
⚠️ GitHub 新仓库默认分支是 main。

7、刷新你的 GitHub Pages 网站

推送成功后：GitHub会自动构建一般需要几十秒 ~ 1分钟，然后你的博客就更新了，访问：
```text
https://你的用户名.github.io
```
也可以去到你的远程仓库访问
