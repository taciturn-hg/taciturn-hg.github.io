    提交描述结构

    一般推荐 简短标题 + 可选详细描述：

    <类型>: <简短描述> <可选详细描述> 类型（Type）：说明这次修改的性质，常用类型： feat：新增功能 / 页面 / 模块 fix：修复问题 docs：只修改文档或 Markdown style：格式修改（缩进、空格、样式） refactor：重构代码或文件结构 chore：杂项（比如更新依赖、修改配置） 简短描述：一句话说明本次修改 详细描述（可选）：写更多背景或修改说明
